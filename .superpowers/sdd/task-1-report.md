## Deviations / Notes

1. **Tailwind v4, not v3.** `create-next-app@latest` ships Tailwind v4 (4.3.1) using
   the new `@tailwindcss/postcss` PostCSS plugin workflow — no `tailwind.config.js`
   is generated. Task 2 (Tailwind theme setup) must use the v4 CSS-first approach
   (`@theme { ... }` in `globals.css`, `@import "tailwindcss"`), NOT the v3
   JS-config approach. Branch accordingly.

2. **Next.js 16.2.9 + React 19.2.4.** Significantly newer than what's in earlier
   training snapshots. The scaffold auto-generated `web/AGENTS.md` and `web/CLAUDE.md`
   with the warning: *"This is NOT the Next.js you know — breaking changes may be
   present."* Read `node_modules/next/dist/docs/` before any Next-API work in later tasks.

3. **`--no-turbopack` flag behavior.** The flag was accepted and applied to the
   generated `dev` script (`"dev": "next dev"`). However, `next build` in Next 16
   uses Turbopack by default regardless (see build log: "Next.js 16.2.9 (Turbopack)").
   This is expected Next 16 behavior, not a deviation from the brief.

4. **create-next-app ran `git init`** in `web/` ("Initialized a git repository").
   The brief explicitly states this is acceptable as long as files, node_modules,
   lint, and build all work — which they do. No commits were made by this task.
   No other git commands were run.

5. **2 moderate severity vulnerabilities** were reported by npm after the dev-deps
   install (consistent across all three install steps). Not investigated — out of
   scope for Task 1 scaffolding; later tasks may run `npm audit` if desired.

6. All deps installed on the first attempt; no flags rejected, no retries needed.
